## プロジェクト概要 <a id="project-overview"></a>

| 項目 | 内容 |
|------|------|
| **ゲームタイトル** | JumpingRabbit |
| **一言で説明すると** | ウサギが空へと飛び跳ねていく縦スクロール型アクションゲーム |
| **ターゲット** | カジュアルに遊べるユーザー |
| **制作の目的** | 2Dの当たり判定・攻撃処理を自分で実装する |
| **ゲームの目的** | ランダム性のある、繰り返し遊べるゲーム |
| **プレイヤー操作** | AD又は矢印キーで移動・スペースでジャンプ・Bで攻撃（ビーム発射） |

|　　　　　　　　|　stage2： ハイスコア・とにかく上を目指す|
| **失敗条件** | ライフ３つ消化（エネミーに衝突あるいは画面下に落下を計３回） |
| **特徴・工夫点** | settingボタンで動作一時停止・音量・スクリーンサイズの調節　|
|                 |→　それらを保存する機能 |
|                 |足場・敵がランダム生成・高く上ることで難易度アップ |

## 発表内容（予定） <a id="presentation-plan"></a>

| 項目 | 内容 |
|------|------|
| **タイトル** | Jumping Rabbit|
| **ゲーム紹介** |
うさぎが上へ向かってジャンプと攻撃を駆使しながら進むゲーム。
ステージ1はゴールあり、ステージ2はゴールなしでひたすらハイスコアを目指して進む
ステージ1はチュートリアル的な位置付けとして、難易度が急激に上がらないよう一定間隔で敵を出現。ステージ2では高度に応じて出現頻度が増加するシステムを実装し、段階的な難易度設計を行った|
| **アピールポイント** | 
足場：プレイヤーの進行に合わせて足場を自動生成し、画面外になった古い足場を削除することで、無限に続くステージのような仕組みを実装した。ランダムな間隔で足場を配置しつつ、X座標の範囲制限を設けることで、遊びやすさとランダム性の両立を意識した。|

敵：プレイヤーの到達高度に応じて敵の出現頻度を変化させる仕組みを実装した。ゲーム序盤は敵の出現数を抑えて操作に慣れやすくし、高度が上がるにつれて出現間隔を短縮することで自然に難易度が高くなる設計。また、複数種類の敵をランダムに生成することで、単調にならないゲーム体験を意識した。

オプション機能の実装：フルスクリーンのON/OFF切り替え機能　/　BGM・SE音量調整機能　/　設定内容の保存機能
・ユーザーが快適にプレイできるよう、オプション画面を実装。UIにはUnityのPanel、Button、Toggle、Sliderを使用し、各種設定をリアルタイムで反映できるようにした。
また、PlayerPrefsを利用して設定情報を保存することで、ゲームを再起動した際にも前回の設定が維持される仕組みを実装している。
<img width="583" height="442" alt="スクリーンショット 2026-06-23 141103" src="https://github.com/user-attachments/assets/8af99bf9-72db-4b0f-92d1-5bef687dd720" />



| **頑張った点・工夫した点** | 
エネミー：敵キャラクターは一定速度で落下する仕組みを実装した。またプレイヤーから十分離れた敵オブジェクトを自動削除することで、不要なオブジェクトが蓄積しないようにし、ゲームのパフォーマンス維持を意識した。
オブジェクト指向を意識し、移動処理とデータ管理(現時点ではかなり小さいが)を別スクリプトに分離することで、保守性と拡張性を向上させました。|

| **苦労した点・エラーと解決** | 
足場：前回のゲーム作成時に、何回も遊べるゲームにしたいと思い今回は足場をランダム設定にした。
→ランダム設定の間隔調節が難しく、得に1ststageでは連続ジャンプ不可のためギリギリ進める位置にランダムで出すのが苦労した。chatgptとも相談し、何回も調整することで解決。
無限背景設定：2ndstageでは無限ステージにするため、[CreateBackground(lastY + backgroundHeight);]で背景をつぎ足し続けることで終わりのない宇宙を表現した。ただ同じ背景をつぎ足しているので単調になってしまう
→左右反転と上下反転を組み合わせることでつなぎ目を自然に、かつ変化をうみだすことができた。|

| **今後追加したいこと** |今後の拡張として、到達高度やスコアに応じて新たな攻撃アクションを解放する成長要素の追加を検討している。また、敵キャラクターごとに移動・攻撃パターンを実装し、ステージ進行に応じて難易度が変化する仕組みを導入することで、ゲーム性の向上を図りたい。 |
