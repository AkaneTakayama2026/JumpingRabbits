## 開発ルール <a id="development-rules"></a>

- **Git運用**
  - 1〜2時間ごとに Commit
  - 1日 2〜3回 Push
  - 作業前に必ず **Pull** して最新を取得
  - Commit メッセージは内容がわかる日本語 or 英語（例：`プレイヤー移動を追加`）

- **Unityプロジェクト設定**
  - Unity 6（URP）を使用
  - 不要アセットは増やしすぎない（軽量管理）
  - `.gitignore` は Unity 公式テンプレートを使用

- **フォルダ構成（推奨）**
```
Assets/
├─ Scripts/
├─ Prefabs/
├─ Scenes/
├─ UI/
└─ Audio/
```

## 発表内容（予定） <a id="presentation-plan"></a>

| 項目 | 内容 |
|------|------|
| **タイトル** | |
| **ゲーム紹介（30秒）** | 

ステージ1はチュートリアル的な位置付けとして、難易度が急激に上がらないよう一定間隔で敵を出現させました。その後のステージでは高度に応じて出現頻度が増加するシステムを実装し、段階的な難易度設計を行いました|

| **アピールポイント** | 
足場：プレイヤーの進行に合わせて足場を自動生成し、画面外になった古い足場を削除することで、無限に続くステージのような仕組みを実装した。ランダムな間隔で足場を配置しつつ、X座標の範囲制限を設けることで、遊びやすさとランダム性の両立を意識した。|

敵：プレイヤーの到達高度に応じて敵の出現頻度を変化させる仕組みを実装した。ゲーム序盤は敵の出現数を抑えて操作に慣れやすくし、高度が上がるにつれて出現間隔を短縮することで自然に難易度が高くなる設計。また、複数種類の敵をランダムに生成することで、単調にならないゲーム体験を意識した。

オプション機能の実装：フルスクリーンのON/OFF切り替え機能　/　BGM・SE音量調整機能　/　設定内容の保存機能
・ユーザーが快適にプレイできるよう、オプション画面を実装しましたUIにはUnityのPanel、Button、Toggle、Sliderを使用し、各種設定をリアルタイムで反映できるようにしました。
また、PlayerPrefsを利用して設定情報を保存することで、ゲームを再起動した際にも前回の設定が維持される仕組みを実装しています。


| **頑張った点・工夫した点** | 
エネミー：敵キャラクターは一定速度で落下する仕組みを実装した。またプレイヤーから十分離れた敵オブジェクトを自動削除することで、不要なオブジェクトが蓄積しないようにし、ゲームのパフォーマンス維持を意識した。
オブジェクト指向を意識し、移動処理とデータ管理(現時点ではかなり小さいが)を別スクリプトに分離することで、保守性と拡張性を向上させました。|
| **苦労した点・エラーと解決** | 
足場：前回のゲーム作成時に、何回も遊べるゲームにしたいと思い今回は足場をランダム設定にした。
→ランダム設定の間隔調節が難しく、得に1ststageでは連続ジャンプ不可のためギリギリ進める位置にランダムで出すのが苦労した。chatgptとも相談し、何回も調整することで解決。
無限背景設定：2ndstageでは無限ステージにするため、[CreateBackground(lastY + backgroundHeight);]で背景をつぎ足し続けることで終わりのない宇宙を表現した。ただ同じ背景をつぎ足しているので単調になってしまう
→左右反転と上下反転を組み合わせることでつなぎ目を自然に、かつ変化をうみだすことができた。|
